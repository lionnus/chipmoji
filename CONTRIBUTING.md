# Contributing to Chipmoji

Thanks for your interest in improving Chipmoji! This project is a curated guide, so the
bar for new entries is intentionally high — but fixes, clarifications, and well-justified
additions are very welcome.

## Ways to contribute

- **Report a problem** — open a [bug report](https://github.com/lionnus/chipmoji/issues/new/choose)
  for wrong descriptions, broken behavior, or typos.
- **Propose an entry** — open a [feature request](https://github.com/lionnus/chipmoji/issues/new/choose)
  describing the hardware intent that existing entries do not cover.
- **Send a pull request** — for fixes and agreed-upon additions.

## Getting set up

Requires [Node.js](https://nodejs.org/) 20+.

```bash
npm install
npm run dev    # local dev server
npm run lint   # lint before pushing
npm run build  # type-check + regenerate the TXT export + build
```

## Editing the chipmoji list

The single source of truth is [`src/data/chipmojis.ts`](./src/data/chipmojis.ts).

- **Never** edit `public/chipmoji-instructions.txt` by hand — it is generated by
  `npm run build` from the data file.
- Keep `type` accurate: use `standard gitmoji` for entries that mirror upstream
  [Gitmoji](https://gitmoji.dev/) meanings, and `chipmoji extension` for hardware-specific
  additions.
- Preserve standard Gitmoji meanings. Do not repurpose an existing Gitmoji for an
  unrelated hardware concept.
- Provide a realistic `example` in the documented commit format:
  `<intention> [scope?]: <message>`.
- Mark an entry `recommended: true` only if it is broadly useful day to day.

## Pull request checklist

- [ ] `npm run lint` passes.
- [ ] `npm run build` passes (this also regenerates the TXT export — commit the result).
- [ ] Changes to the list were made in `src/data/chipmojis.ts`.
- [ ] Commit messages follow the Chipmoji format (eat your own dog food! 🐍).
- [ ] The PR description explains the *intent* behind any new entry.

## Commit style

Use the Chipmoji convention itself:

```text
<intention> [scope?]: <message>
```

For example:

```text
:sparkles: data: add chipmoji for power-domain crossings
:memo: readme: clarify the TXT export workflow
```
